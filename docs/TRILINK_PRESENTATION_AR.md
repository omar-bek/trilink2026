# شريحة 3: الحل

## TriLink: مشتريات رقمية من الألف إلى الياء

> منصة واحدة تأخذ احتياج الشراء من **الطلب → عرض الأسعار → المناقصة → العقد → الدفع → الشحن** مع امتثال تنظيمي كامل مدمج.

### ما يميّزنا:

```
┌─────────────────────────────────────────────────────────┐
│                                                         │
│  1️⃣  امتثال إماراتي أصيل                               │
│      (مش أداة غربية معدّلة للمنطقة)                     │
│                                                         │
│  2️⃣  ذكاء اصطناعي مدمج في المشتريات                    │
│      (تحليل مخاطر، تنبؤ أسعار، مساعد تفاوض)           │
│                                                         │
│  3️⃣  معمارية متعددة الأدوار (13 دور)                    │
│      (مشتري، مورد، لوجستيك، حكومة، إدارة...)          │
│                                                         │
│  4️⃣  لوحة رقابة حكومية                                  │
│      (تقارير المنافسة، التواطؤ، ESG، العقوبات)         │
│                                                         │
│  5️⃣  ثنائي اللغة مع دعم RTL                             │
│      (عربي/إنجليزي بتبديل فوري)                        │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

# شريحة 5: مسار العمل الأساسي

## من الطلب إلى التسليم

```
    ┌──────────────────┐
    │   طلب الشراء     │  ← المشتري ينشئ طلب شراء داخلي
    │  Purchase Request │
    └────────┬─────────┘
             │
             ▼
    ┌──────────────────┐
    │  موافقة المدير   │  ← المدير يراجع ويوافق/يرفض
    │ Manager Approval  │     (دعم الموافقة الجماعية)
    └────────┬─────────┘
             │
             ▼
    ┌──────────────────┐
    │  إنشاء تلقائي    │  ← النظام يحوّل PR إلى RFQ تلقائياً
    │   لعرض الأسعار   │
    │   Auto-RFQ        │
    └────────┬─────────┘
             │
             ▼
    ┌──────────────────┐     ┌──────────────────┐
    │  عروض الموردين   │◄────│    مفاوضات /     │
    │  Supplier Bids    │     │  مزادات حية      │
    └────────┬─────────┘     │  Negotiations /   │
             │               │  Live Auctions    │
             ▼               └──────────────────┘
    ┌──────────────────┐
    │  قبول العرض      │  ← تقييم، مقارنة، ترجيح ICV
    │  Bid Acceptance   │
    └────────┬─────────┘
             │
             ▼
    ┌──────────────────┐
    │  إنشاء العقد     │  ← عقد تلقائي من العرض المقبول
    │  Auto-Contract    │     (إصدارات، تعديلات، ملاحظات)
    └────────┬─────────┘
             │
             ▼
    ┌──────────────────┐
    │  التوقيع الرقمي  │  ← 3 درجات: بسيط / UAE Pass / TSP
    │ Digital Signing   │     (شهادة تدقيق التوقيع)
    └────────┬─────────┘
             │
             ▼
    ┌──────────────────┐
    │  الدفع           │  ← Stripe / PayPal / Escrow
    │  Payment          │     (فاتورة ضريبية تلقائية)
    └────────┬─────────┘
             │
             ▼
    ┌──────────────────┐
    │  الشحن والتتبع   │  ← Aramex / DHL / FedEx / UPS
    │  Shipment         │     (GPS + جمارك + Dubai Trade)
    └────────┬─────────┘
             │
             ▼
    ┌──────────────────┐
    │  إتمام العقد     │  ← تقييم + إعادة طلب
    │  Completion       │
    └──────────────────┘

              │ مشتري  │ │ مورد   │  │ │  مبيعات     │ │
              │        │ │        │  │ └─────────────┘ │
              │• طلبات │ │• عروض  │  └────────────────┘
              │  شراء  │ │  أسعار │
              │• RFQs  │ │• منتجات│
              │• عقود  │ │• عقود  │
              └────────┘ └────────┘
```

### نظام الصلاحيات:
- **Spatie Laravel Permission** — صلاحيات دقيقة على مستوى القدرات
- **قائمة صلاحيات لكل مستخدم** — ليس فقط حسب الدور
- **حراسة على مستوى المسارات** — Middleware للتحقق قبل الوصول
- **نطاق الشركة** — كل استعلام محدود بـ company_id المستخدم
